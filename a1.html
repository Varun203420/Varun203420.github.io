// HCDE 439 – A1: Independent LED blinks with non-blocking timers (millis())
// Pins: D8 (LED1), D9 (LED2), D10 (LED3)
const byte LED_PINS[3] = {8, 9, 10};
unsigned long intervals[3] = {300, 700, 1130};
unsigned long lastToggle[3] = {0, 0, 0};
bool stateLED[3] = {false, false, false};

void setup() {
  for (byte i = 0; i < 3; i++) {
    pinMode(LED_PINS[i], OUTPUT);   // set pin to output mode
    digitalWrite(LED_PINS[i], LOW); // ensure LED starts off
  }
}

void loop() {
  unsigned long now = millis();     // non-blocking time base
  for (byte i = 0; i < 3; i++) {
    if (now - lastToggle[i] >= intervals[i]) {
      stateLED[i] = !stateLED[i];                // flip state
      digitalWrite(LED_PINS[i], stateLED[i]);    // write to pin
      lastToggle[i] = now;                       // remember toggle time
    }
  }
}

Answers to additional questions

  1. Q1: See the figure above: three square waves at each LED’s Vf (≈2.0 V red, 2.2 V green, 3.0 V blue) with periods 0.6 s, 1.4 s, and 2.26 s respectively; ~50% duty.
  2. Q2: About 18 practical independent outputs on an Uno (avoid D0/D1): D2–D13 plus A0–A5 as digital. At ~10 mA each that’s ≈ 180 mA total; stay within per-pin (~20 mA max recommended) and overall current limits. For many LEDs, use 5–8 mA each or external drivers.
  3. Q4: Blinking appears steady around 50–60 Hz (period ≤ ~16–20 ms) under typical lighting—near the flicker-fusion threshold.
  4. Q5 (AI tools): I used ChatGPT to plan resistor values with Ohm’s Law, write non-blocking code using millis(), generate the V–t figure, and structure this page. I verified values and tested on my Uno.
